# Отчёт о качестве данных

## 1. Введение

**Датасет:** Набор данных для анализа рынка подержанных автомобилей BMW
**Источник:** [\[Ссылка на источник\]](https://www.kaggle.com/datasets/algozee/bmw-dataset)  
**Размер:** [10781 Х 9]  

**Предметная область:** Датасет содержит информацию об автомобилях марки BMW, выставленных на продажу на вторичном рынке. Включает такие характеристики, как модель, год выпуска, пробег, тип топлива, коробка передач, цена, налог, объем двигателя

**Условный заказчик:** Команда аналитиков крупного онлайн-автомаркета (например, Auto.ru) или отдел стратегического планирования сети дилеров BMW.  

**Возможные задачи ИАД:**
1. **Прогнозирование рыночной цены** автомобиля на основе его характеристик (регрессия).
2. **Сегментация автомобилей** для выделения групп с похожими параметрами и ценовым позиционированием (кластеризация).
3. **Поиск аномалий** в объявлениях (например, мошеннические предложения с нереально низкой ценой).

## 2. Паспорт датасета

| Столбец | Тип данных | Описание | Примечания |
|---------|------------|----------|------------|
| model | object | Модель автомобиля BMW | Категориальный признак. Примеры: '320d', 'X5', 'i3' |
| year | int64 | Год выпуска автомобиля | Числовой признак. Диапазон: 1996-2020 |
| price | int64 | Цена автомобиля в фунтах стерлингов (£) | Числовой признак. Диапазон: £1,500 - £123,456 |
| transmission | object | Тип коробки передач | Категориальный признак. Значения: 'Automatic', 'Manual', 'Semi-Auto' |
| mileage | int64 | Пробег автомобиля в милях | Числовой признак. Диапазон: 1 - 323,000 миль |
| fuelType | object | Тип топлива | Категориальный признак. Значения: 'Petrol', 'Diesel', 'Hybrid', 'Electric' |
| tax | int64 | Годовой дорожный налог в фунтах | Числовой признак. Диапазон: £0 - £600 |
| mpg | float64 | Расход топлива (миль на галлон) | Числовой признак. Диапазон: 14.2 - 217.3 mpg |
| engineSize | float64 | Объем двигателя в литрах | Числовой признак. Диапазон: 1.0 - 6.6 л |

## 3. Основные проблемы качества данных

| № | Тип проблемы | Описание (где и что) | Потенциальный риск для анализа | Предлагаемое решение / стратегия |
|---|--------------|----------------------|--------------------------------|-----------------------------------|
| 1 | Пропуски | В столбце `engineSize` обнаружено X% пропущенных значений. | Невозможность использовать этот признак в модели без обработки. Искажение статистик. | 1. Заполнить медианой по аналогичным моделям/году. 2. Удалить строки, если пропусков немного. 3. Добавить бинарный признак "пропуск engineSize". |
| 2 | Выбросы | В столбце `price` обнаружены экстремально высокие значения (свыше Z единиц), не характерные для рынка. | Могут сильно исказить результаты регрессионного анализа и завысить ошибку модели. | 1. Применить логарифмирование цены. 2. Усечь значения на уровне 99-го перцентиля. 3. Исследовать природу выбросов — возможно, это новые модели, которые требуют отдельного анализа. |
| 3 | "Грязные" категории | В столбце `transmission` встретились значения "Automatic", "automatic", "AUTO". | При группировке или one-hot encoding эти значения будут считаться разными категориями, что приведёт к ошибкам. | Привести все значения к единому регистру (например, нижнему) и заменить синонимы. |
| 4 | Невозможные значения | В столбце `year` найдено значение "1900" (или 2030). | Искажение статистик и моделей, основанных на возрасте автомобиля. | Удалить строки с годами вне реалистичного диапазона (например, 1990–2023). |
| 5 | Дубликаты | Обнаружено N полностью повторяющихся строк. | Завышение значимости одних и тех же наблюдений, искажение статистик. | Удалить полные дубликаты, оставив первую запись. |

## 4. План улучшения качества

**Приоритет 1 (критично):**    
*Очистка категориальных признаков:* Стандартизация регистра и значений в столбцах `transmission`, `fuelType`.  
**Приоритет 2:**         
*Удаление полных дубликатов.*            
**Приоритет 3:**  
*Обогащение данных:* При возможности добавить внешние данные (например, индекс надёжности модели, среднюю стоимость страхования).

## 5. Краткое резюме

В ходе первичного исследования датасета BMW были выявлены типичные для сырых рыночных данных проблемы: пропуски в ключевых технических характеристиках, выбросы в ценах, неконсистентность в категориальных признаках. Наибольшее внимание на следующем этапе следует уделить очистке и согласованию категориальных данных, а также стратегии обработки выбросов в цене, так как это напрямую повлияет на качество любой прогнозной модели. Датасет обладает высоким потенциалом для решения бизнес-задач, связанных с ценообразованием и сегментацией автомобилей, после выполнения описанных шагов по улучшению качества.
